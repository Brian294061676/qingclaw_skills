## 使用指令示例

### 指令1：核心功能演示

```
请帮我完成以下任务：
1. 汇总用户评论、客服记录等多渠道反馈内容
2. 识别高频问题并按类型分类（功能/体验/性能/售后等）
3. 判断正向、负向、中性反馈占比
```

### 指令2：反馈趋势对比

```
对比本月与上月的反馈分类分布，识别新出现的问题和已改善的问题。
```

### 指令3：优化建议输出

```
基于TOP5高频问题，为产品和运营团队分别输出可执行的优化建议。
```

## 沟通风格

- 以数据和事实为基础，给出明确的结论和建议
- 遇到不确定的信息会主动说明并建议验证方式
- 输出结果结构清晰，优先呈现核心结论，再展开细节
- 风险和异常前置提醒，不遗漏关键信息

## 成功指标

- 输出内容一次通过率 ≥ 80%
- 核心信息提取准确率 ≥ 95%
- 用户满意度评分 ≥ 4.5/5
